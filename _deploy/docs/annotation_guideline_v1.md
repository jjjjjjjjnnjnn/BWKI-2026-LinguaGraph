# Annotation Guideline v1.0

CognitiveSpace — 教师/人工标注规范

---

## 目标

为每条学生回答标注：
1. **Concepts** — 学生提到了哪些概念
2. **Relations** — 概念之间的关系
3. **Missing Hints** — 学生应该提到但没提到的连接

---

## 概念标注规则

### 什么是概念？

领域术语，不是普通词汇。

| ✅ 是概念 | ❌ 不是概念 |
|-----------|-----------|
| 导数 | 计算 |
| 极限 | 知道 |
| 积分 | 觉得 |
| 变化率 | 很难 |

### 命名规则

- 用中文命名（德语/英语回答也标注中文）
- 如果德语术语是标准用法，保留德语（如 Ableitung）
- 一个概念只出现一次

### 示例

输入：「导数表示变化率，积分是导数的逆运算」
标注：`["导数", "变化率", "积分"]`

## 关系标注规则

### 关系类型（封闭集合）

| 类型 | 定义 | 示例 |
|------|------|------|
| `prerequisite` | A 是学习 B 的前提 | 极限 → 导数 |
| `represents` | A 代表/表示 B | 导数 → 变化率 |
| `inverse_of` | A 是 B 的逆运算 | 积分 → 导数 |
| `is_part_of` | A 是 B 的一部分 | 极限 → 微积分 |
| `has_part` | A 包含 B | 微积分 → 微分 |
| `requires` | A 需要 B 才能定义 | 导数 → 极限 |
| `determines` | A 决定 B | 二阶导数 → 凹凸性 |
| `implies` | A 蕴含 B | 可导 → 连续 |

### 关系方向

`from → to` 表示 "from 是 to 的 prerequisites/represents/..."

示例：
```json
{"from": "极限", "to": "导数", "type": "prerequisite"}
```
含义：极限是导数的前提

### 标注置信度

每条关系必须标注 confidence（0-1）：

| confidence | 含义 |
|-----------|------|
| 1.0 | 确定正确 |
| 0.8 | 很可能正确 |
| 0.6 | 不太确定 |

## Missing Hints 标注规则

### 什么是 Missing Hint？

学生**应该提到但没提到**的概念连接。

### 标注条件

- 只标注专家认为重要的连接
- 不标注"学生说错"的内容
- 只标注"学生没说"的内容

### 标注格式

```json
{
  "from": "极限",
  "to": "导数",
  "reason": "学生不知道极限是导数的基础",
  "confidence": 0.9
}
```

## 完整标注示例

### 输入

「我知道导数表示变化率，但不知道为什么需要极限。」

### 标注

```json
{
  "concepts": ["导数", "变化率", "极限"],
  "relations": [
    {"from": "导数", "to": "变化率", "type": "represents", "confidence": 1.0}
  ],
  "missing_hints": [
    {
      "from": "极限",
      "to": "导数",
      "reason": "学生不知道为什么需要极限来定义导数",
      "confidence": 0.9
    }
  ]
}
```

## 标注流程

1. 阅读学生回答
2. 标注 Concepts
3. 标注 Relations
4. 标注 Missing Hints
5. 检查一致性

## 一致性检查

- 两名标注员各标 10 条
- 计算 Cohen's Kappa
- 目标：Kappa > 0.7
- 如果 Kappa < 0.7，讨论分歧，修订规范
